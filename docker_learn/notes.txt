#### DOCKER compose [multiple containers SETUP, CREATE, DESTROY]
in real world we need to run multiple containers simultaneously like 
seperate for (postgres, redis, mailhog, kafka )all with diff PORTS in a single project so  we need a file to write pipeline forthis.


now make a [docker-compose.yaml] file write pipeline and configuration there and run by this cmd :
>> docker compose up

[now we can see in docker desktop one stack having 2 containers running]

cmd >> docker compose down [removes whatevr the UP command made]

cmd : docker compose up -d [detached mode run in bg no need of active terminal and can also use DOWN cmd to stop/remove]
